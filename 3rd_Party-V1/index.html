3rd Party files

Process: We receive files from a vendor that is our competitor but due to legal reasons they have to send us the requested data. The data houses information relating to shareholders that we need to upload to our database.

Challenge:

The process is manual, each file is password protected, taking 2 to 5 days to complete due to competing priorities, the value of the data can range from 2,500 to 80,000 rows per a file.

Solution:

Using Excel, VBA and power query this process was streamlined to be completed in 15 minutes. A time savings of 2 to 5 days of manual work.

Old process of completing the task - If you would like to skip ahead to the part that I improved, please click here.

Details of the Old Process:

The file(s) aresent via email by the vendor to a specific mailbox that is monitored by the team. The mailbox is checked daily for new files. Once a new file is identified, the file is downloaded and saved to a local drive.

Next, the file is opened in Microsoft Excel and the password is entered to unlock the sheet(s). This password is obtained from a secured database that houses all vendor passwords. Once the sheet(s) are unlocked, they are consolidated into one sheet using formulas and manual copy/paste.

The data is then filtered for specific criteria, formatted, and uploaded to the database using SQL. The process is repeated for each file.

This process can take 2 to 5 days depending on competing priorities and the number of rows of data per file.

Details of the New Process:

A new Excel file was created that uses VBA and power query to automate the process. The new file is opened and the user is prompted to select the files that need to be processed. Once the files are selected, the VBA code unlocks the sheets and consolidates them into one sheet.

The data is then automatically filtered, formatted, and uploaded to the database using SQL. The entire process takes 15 minutes, a time savings of 2 to 5 days of manual work.
